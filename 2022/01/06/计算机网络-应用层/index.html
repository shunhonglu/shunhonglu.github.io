Web 应用

1. URL

URL(Uniform Resource Locator，统一资源定位符)

2. 连接的类型

  • 非持久性连接（Nonpersistent HTTP）
  • 持久性连接（Persistent HTTP）

3. HTTP 请求消息

1. 请求消息通用格式


2. HTTP 方法

  • GET方法：获取资源。
  • HEAD方法：请 Server 不要将所请求的对象放入响应消息中。
  • POST方法：在请求消息的信息体（entity body）中上传客户端的输入。
  • PUT方法：将消息体中的文件上传到URL字段所指定的路径。
  • DELETE方法：删除 URL 字段所指定的文件。

使用 GET 方法，输入信息也可以通过 request 行的 URL 字段上传。

5. Cookie 技术

HTTP 协议是无状态的。

某些网站为了辨别用户身份，进行 session 跟踪而储存在用户本地终端上的数据（通常经过加密）。

1. Cookie 的组件

  • HTTP 响应消息的 Cookie 头部行
  • HTTP 请求消息的 Cookie 头部行
  • 保存在客户端主机上的 Cookie 文件，由浏览器管理
  • Web 服务器端的后台数据库

Email 应用

1. SMTP 协议

SMTP 协议端口号为25，采用异步通信方式。Email 消息只能包含七位 ASCII 码，可以采用 MINE 进行拓展。MINE 并没有改动或者取代 SMTP，而是定义了非 ASCII 码的编码规则。

2. POP3 协议

3. IMAP 协议

IMAP 协议中客户端和服务器上的邮件保持同步，如果不手动删除邮件，那么服务器上的邮件也不会被删除。IMAP 这种做法可以让用户随时随地去访问服务器上的邮件。
