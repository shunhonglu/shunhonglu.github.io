计算机网络-传输层

多路复用/分用

  • 多路分用：传输层依据头部信息将收到的 Segment 交给正确的 Socket，即不同的进程。
  • 多路分用：从多个 Socket 接收数据，为每块数据封装上头部信息，生成 Segment，交给网络层。

可靠数据传输

1. GBN 协议

发送方：

  1. 单个定时器
  2. 累计确认
  3. 若某个报文段未确认，重传序列号大于等于 n，还未收到 ACK 的所有分组。

接收方：

  1. 无缓存，直接丢弃乱序到达的分组
  2. 发送拥有最高序列号的，已被正确接收的分组的 ACK。

2. SR 协议

发送方：

  1. 每个分组设置定时器
  2. 单个确认
  3. 重传未收到 ACK 的分组

接收方：

  1. 设置缓存，缓存乱序达到的分组
  2. 接收方返回的 ACK 是当前接收成功报文段的序号

3. TCP 协议

发送方：

  1. 单个定时器
  2. 累计确认
  3. 超时时重传序列号最小的还未确认的段

接收方：

  1. 有缓存
  2. TCP 中的 ACK 是指接收端希望从发送端收到的下一字节的序号

拥塞控制

1. 拥塞控制原理

可靠数据传输和拥塞控制的区别？
可靠数据传输是针对个体利益，而拥塞控制则是针对群体利益。

拥塞控制和流量控制的区别？
流量控制针对的是接收方处理不了，而拥塞控制针对的是网络处理不了。

2. TCP 拥塞控制

1. 限制发送速率

$$LastByteSent-LastByteAcked \leq CongWin$$

$$rate \approx {CongWin \over RTT}$$

2. 感知网络拥塞

Loss 事件 = timeout 或3个重复ACK。

3. 如何调整发送速率

  • 加性增（Additive Increase）：每个 RTT 将 CongWin 增大一个 MSS。
  • 乘性减（Multiplicative Decrease）：发生 Loss 后将 CongWin 减半。
  • 慢启动：当连接开始时（CongWin = 1），指数性增长。
  • Loss 事件发生时，Threshold 被设为 Loss 事件前 CongWin 值的1/2。当发生 timeout 时，CongWin 直接设为 1 个 MSS。当收到 3 个重复 ACKs时，CongWin 切到一半。

UDP

UDP 基于 Internet IP 协议。提供的服务有复用/分用，简单的错误检验。

UDP 无连接，UDP 发送方和接收方之间不需要握手，并且每个 UDP 段的处理独立于其他段。

2. UDP 校验和

  • 发送方：将段的内容视为 16-bit 整数，计算所有整数的和，进位加在和的后面，将得到的值按位取反，得到校验和。发送方将校验和放入校验和字段。
  • 接收方：计算所收到段的校验和，将其与校验和字段进行对比，若不相等则检测出错误，若相等则没有检测出错误（但可能有错误）。

注意理解 TCP 中的确认号！

确认号的意思是发送方在发送数据包（假设大小为 10 byte）时，同时送上一个序号（假设为 500），那么接收方收到这个数据包以后，就可以回复一个确认号（510 = 500 + 10）告诉发送方“我已经收到了你的数据包，你可以发送下一个数据包，序号从 510 开始”。
