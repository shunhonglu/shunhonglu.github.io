IP 分片

  • 标识字段占 16 位：标识一个 IP 分组
  • 标志位字段占 3 位
    • DF (Don’t Fragment)
    • MF (More Fragment)
  • 片偏移字段占 13 位：一个 IP 分组分片封装原 IP 分组数据的相对偏移量

CIDR 与路由聚集

无类域间路由（CIDR：Classless InterDomain Routing）
无类地址格式：a.b.c.d/x，其中x为前缀长度。

DHCP 协议

动态主机配置协议（DHCP：Dynamic Host Configuration）
DHCP 协议在应用层实现。

  • 主机广播“DHCP discover”（发现报文）
  • DHCP 服务器利用“DHCP offer”（提供报文）进行响应
  • 主机请求 IP 地址：“DHCP request”（请求报文）
  • DHCP 服务器分配 IP 地址：“DHCP ack”（确认报文）

主机请求 IP 地址时同样需要广播，目的是告诉网络中的其他 DHCP 服务器“我”已经确定从这个 DHCP 服务器申请 IP 地址，其他服务器可以收回预分配的地址。

IPv6

1. IPv6 数据报格式


  • 固定长度的 40 字节基本首部
  • 不允许分片

2. IPv6 地址表示形式

  • 16 位一组，每组用对应的 16 进制数表示
  • 压缩形式
  • IPv4 嵌入形式
  • 地址前缀（IPv6 不再使用掩码）

3. IPv4 向 IPv6 过渡

隧道：IPv6 数据报作为 IPv4 数据报的载体进行封装，穿越 IPv4 网络。
